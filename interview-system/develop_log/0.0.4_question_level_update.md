# 版本 0.0.4 - 题目等级体系重构

## 更新内容

根据新的等级评定标准，重构了题目等级体系：

### 1. 新的等级体系

#### 基础等级（面试环节）
- **初级**：基本程序编写能力，能完成简单编程任务
- **中级**：独立完成中等复杂度任务，具备故障排除能力，对科研有一定了解
- **高级**：复杂算法和系统设计，深入科研能力，较强沟通协作能力

#### 专精等级（仅三等，面试环节）
- **专精三等**：配合以下专精类型
  - 算法与程序设计
  - 系统设计
  - 商业设计
  - 科研

> 注：专精二等和一等通过简历提交和现场答辩评定，不纳入系统面试环节

### 2. 模型变更 (Question.java)

#### 新增枚举
```java
// 题目等级
enum QuestionLevel {
    BASIC("初级", "基础等级", 1, false),
    INTERMEDIATE("中级", "基础等级", 2, false),
    ADVANCED("高级", "基础等级", 3, false),
    SPECIALIZATION_THREE("专精三等", "专精等级", 4, true)
}

// 专精类型
enum SpecializationType {
    NONE, ALGORITHM, SYSTEM_DESIGN, BUSINESS, RESEARCH
}
```

#### 字段变更
- 移除：`difficulty` 字段
- 新增：`level` (QuestionLevel) 和 `specialization` (SpecializationType)

### 3. 数据库变更

新增字段：`specialization VARCHAR(30) DEFAULT 'NONE'`

### 4. 界面更新

#### 题目编辑对话框
- 等级选择：初级/中级/高级/专精三等
- 选择"专精三等"时显示专精类型选择
- 添加等级说明提示

#### 题目浏览/抽取
- 筛选条件更新为新的等级体系
- 表格显示等级时自动组合显示（如："专精三等 - 算法与程序设计"）

## 数据结构对照

| 旧系统 | 新系统 |
|--------|--------|
| EASY (初级) | BASIC (初级) |
| MEDIUM (中级) | INTERMEDIATE (中级) |
| HARD (高级) | ADVANCED (高级) |
| EXPERT (资深) | 移除 |
| - | SPECIALIZATION_THREE (专精三等) |

## 构建运行

```bash
cd interview-system
mvn clean package -s settings.xml -DskipTests
java -jar target/interview-system-1.0-SNAPSHOT.jar --db-password 1234
```

## 注意事项

1. 数据库需要重新初始化或手动添加 `specialization` 字段
2. 旧数据中的 EXPERT 等级题目将被映射为 ADVANCED
